					<h1 class="entry-title">Web Scraping Tutorial &#8212; episode #1 &#8212; Scraping a Webpage (with Bash)</h1>															
				</div>

<span class="vcard author post-author">Written by <span class="fn">Tomi Mester </span>on January 28, 2020</span>

<span>
<p class="last-updated">Last updated on February 06, 2020</p>
				</span>


 <!-- /post-header -->



				    
			    <div class="post-content">
			    
			    	
<p>Running hobby projects is the best way to practice data science before getting your first job. And one of the best ways to get real data for a hobby project is: <strong>web scraping</strong>.</p>



<p>I&#8217;ve been promising this for a long time to <a href="https://data36.com/jds" data-wpel-link="internal">my course participants</a> &#8211; so here it is: my web scraping tutorial series for aspiring data scientists!&nbsp;</p>



<p>I&#8217;ll show you step by step how you can:</p>



<ol><li>scrape a public html webpage</li><li>extract the data from it</li><li>write a script that automatically scrapes <em>thousands</em> of public html webpages on a website</li><li>create useful (and fun) analyses from the data you get</li><li>analyze a huge amount of text</li><li>analyze website metadata</li></ol>



<p>This is episode #1, where we will focus on step #1 (scraping a public html webpage). And in the upcoming episodes we will continue with step #2, #3, #4, #5 and #6 (scaling this up to thousands of webpages, extracting the data from them and analyzing the data we get).</p>



<p>Even more, I&#8217;ll show you the whole process in two different data languages, too, so you will see the full scope. In this article, I will start with the simpler one: <strong>bash</strong>. And in future articles, I&#8217;ll show you how to do similar things in <strong>Python</strong>, as well.</p>



<p>So buckle up! Web scraping tutorial episode #1 &#8212; here we go!</p>



<h2><strong>Before we start&#8230;</strong></h2>



<p>This is a hands-on tutorial. I highly recommend doing the coding part with me (and doing the exercises at the end of the articles).</p>



<p>I presume that you have some bash coding knowledge already &#8212; and that you have your own data server set up already. If not, please go through these tutorials first:</p>



<ol><li><a href="https://data36.com/data-coding-101-install-python-sql-r-bash/" data-wpel-link="internal">How to install Python, R, SQL and bash to practice data science!</a></li><li><a href="https://data36.com/data-coding-101-introduction-bash/" data-wpel-link="internal">Introduction to Bash episode #1</a></li><li><a href="https://data36.com/bash-intro-2-data-coding-101/" data-wpel-link="internal">Introduction to Bash episode #2</a></li><li><a href="https://data36.com/command-line-data-awk-sed-join-date/" data-wpel-link="internal">Introduction to Bash episode #6</a></li></ol>


<span style="display:none" class="tl-placeholder-f-type-shortcode_4225"></span>



<h2><strong>The project: scraping TED.com and analyze talks</strong></h2>



<p>When you run a <a href="https://data36.com/data-science-projects-for-boosting-your-resume/" data-wpel-link="internal">data science hobby project</a>, you should always pick a topic that you are passionate about.</p>



<p>As for me: I love public speaking.</p>



<p>I like to practice it myself and I like to listen to othersâ€¦ So watching TED presentations is also a hobby for me. Thus I&#8217;ll go ahead and analyze TED presentations in this tutorial.&nbsp;</p>



<p>Luckily, almost if not all TED presentations are already available online.</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="503" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-talks-2-1024x503.png" alt="web scraping tutorial - ted talks" class="wp-image-4657" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-talks-2-1024x503.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-talks-2-300x147.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-talks-2-768x377.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-talks-2-973x478.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-talks-2-508x249.png 508w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>&#8230;<br>Even more, their transcripts are available, too!<br>&#8230;</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="549" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2-1024x549.png" alt="web scraping tutorial - ted transcript 2" class="wp-image-4667" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2-1024x549.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2-300x161.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2-768x412.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2-973x521.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2-508x272.png 508w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-ted-transcript-2.png 1702w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>(Thank you TED!)</p>



<p>So we&#8217;ll &#8220;just&#8221; have to write a bash script that collects all those transcripts for us and we can start our in-depth text analysis.</p>



<p><em>Note 1: I picked scraping TED.com just for the sake of example. If you are passionate about something else, after finishing these tutorial articles, try to find a web scraping project that resonates with you! Are you into finance? Try to scrape stock market news! Are you into real estate? Then scrape real estate websites! Are you a movie person? Your target could be imdb.com (or something similar)!</em></p>



<h2><strong>Login to your remote server!</strong></h2>



<p>Okay, so one more time: for this tutorial, you&#8217;ll need <a href="https://data36.com/data-coding-101-install-python-sql-r-bash/" data-wpel-link="internal">a remote server</a>. If you haven&#8217;t set one up yet, now is the time! ðŸ™‚&nbsp;</p>



<p><em>Note: If &#8212; for some reason &#8212; you don&#8217;t like my server setup, you can use a different environment. But I strongly advise against it. Using the exact same tools that I use will guarantee that everything you read here will work on your end, too. So one last time: use <a href="https://data36.com/data-coding-101-install-python-sql-r-bash/" data-wpel-link="internal">this remote server setup</a> for this tutorial.</em></p>



<p>Okay, let&#8217;s say that you have your server. Great!<br>Now open Terminal (or Putty) and log in with your username and IP address.</p>



<p>If everything&#8217;s right, you should see the command lineâ€¦ Something like this:</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="426" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal-1024x426.png" alt="web scraping tutorial 1 - login terminal" class="wp-image-4660" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal-1024x426.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal-300x125.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal-768x320.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal-973x405.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal-508x211.png 508w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-1-login-terminal.png 1456w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<h2><strong>Introducing your new favorite command line tool: curl</strong></h2>



<p>Interestingly enough, in this whole web scraping tutorial, you will have to learn only <em>one</em> new bash command. And that&#8217;s <code>curl</code>.</p>



<p><code>curl</code> is a great tool to access a website&#8217;s whole html code from the command line. <em>(It&#8217;s good for many other server-to-server data transfer processes &#8212; but we won&#8217;t go there for now.)</em></p>



<p>Let&#8217;s try it out right away!</p>



<p><code>curl https://www.ted.com/</code></p>



<p>The result is:</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="654" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-1024x654.png" alt="web scraping tutorial 2 - curl" class="wp-image-4661" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-1024x654.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-300x192.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-768x491.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-973x622.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-508x325.png 508w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl.png 1446w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>Oh boyâ€¦ What&#8217;s this mess??</p>



<p>It&#8217;s the full html code of TED.com &#8212; and soon enough, I&#8217;ll explain how to turn this into something more meaningful. But before that, something important.</p>



<p>As you can see, to get the data you want, you&#8217;ll have to use that exact URL where the website content is located &#8212; and the full version of it. So, for instance this short form won&#8217;t work:</p>



<p><code>curl ted.com</code></p>



<p>It just doesn&#8217;t return anything:</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="213" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url-1024x213.png" alt="web scraping tutorial 2 - curl full url" class="wp-image-4662" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url-1024x213.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url-300x62.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url-768x160.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url-973x202.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url-508x106.png 508w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-2-curl-full-url.png 1452w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>And you&#8217;ll get similar empty results for these:</p>



<p><code>curl www.ted.com</code></p>



<p><code>curl http://www.ted.com</code></p>



<p>Even when you define the <code>https</code> protocol properly but you miss the <code>www</code> part, you&#8217;ll get a short error message that your website content <em>&#8220;has been moved&#8221;</em>:</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="290" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently-1024x290.png" alt="web scraping tutorial 3 - 301 moved permanently" class="wp-image-4663" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently-1024x290.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently-300x85.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently-768x218.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently-973x276.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently-508x144.png 508w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-3-301-moved-permanently.png 1454w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>curl, 301 Moved Permanently</em></figcaption></figure>



<p>So make sure that you type the full URL and you use the one where the website is actually located. This, of course, differs from website to website. Some use the <code>www</code> prefix, some don&#8217;t. Some still operate under the <code>http</code> protocol &#8212; most (luckily) use <code>https</code>.</p>



<p>A good trick to find the URL you need is to open the website in your browser &#8212; and then simply copy-paste the full URL from there into your Terminal window:</p>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="263" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-4-copy-paste-full-url-1024x263.png" alt="web scraping tutorial 4 - copy paste full url" class="wp-image-4664" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-4-copy-paste-full-url-1024x263.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-4-copy-paste-full-url-300x77.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-4-copy-paste-full-url-768x197.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-4-copy-paste-full-url-973x250.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-4-copy-paste-full-url-508x131.png 508w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>So in TED&#8217;s case, it will be this:</p>



<p><code>curl https://www.ted.com</code></p>



<p>But, as I said, I don&#8217;t want to scrape the TED.com home page.</p>



<p>I want to scrape the transcripts of the talks. So let&#8217;s see how to do that!</p>



<h2><strong>curl in action: downloading one TED talk&#8217;s transcript</strong></h2>



<p>Obviously, the first step in a web scraping project is always to find the right URL for the webpage that you want to download, extract and analyze.</p>



<p>For now, let&#8217;s start with one single TED talk. (Then in the next episode of this tutorial, we&#8217;ll scale this up to all 3,300 talks.)</p>



<p>For prototyping my script, I chose the most viewed speech &#8212; which is Sir Ken Robinson&#8217;s &#8220;Do schools kill creativity.&#8221; <em>(Excellent talk, by the way, I highly recommend watching it!)</em></p>



<p>The transcript itself is found under this URL:</p>



<pre class="wp-block-preformatted">https://www.ted.com/talks/sir_ken_robinson_do_schools_kill_creativity/transcript</pre>



<p>So you&#8217;ll have to copy-paste this to the command line &#8212; right after the <code>curl</code> command:</p>



<pre class="wp-block-preformatted">curl https://www.ted.com/talks/sir_ken_robinson_do_schools_kill_creativity/transcript</pre>



<figure class="wp-block-image"><img loading="lazy" width="1024" height="651" src="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript-1024x651.png" alt="web scraping tutorial 5 -- scraping the transcript" class="wp-image-4669" srcset="https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript-1024x651.png 1024w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript-300x191.png 300w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript-768x488.png 768w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript-973x619.png 973w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript-508x323.png 508w, https://data36.com/wp-content/uploads/2020/01/web-scraping-tutorial-5-scraping-the-transcript.png 1450w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>Great!<br>We got our messy html code again &#8212; but this actually means that we are one step closer.</p>

<p>We&#8217;ll try to find a pattern that&#8217;s similar for every single transcript webpage on TED.com and that clearly borders where the useful part of our content (the actual transcript of the talk) starts and ends.</p>



<p>It&#8217;s time for data discovery!<br>Let&#8217;s download a few more talks, compare them and figure out what these patterns could be:</p>

<p>In <a href="https://data36.com/command-line-data-awk-sed-join-date/" data-wpel-link="internal">one of my previous bash tutorials</a>, I have already written about a great command line tool: <code>sed</code>.</p>



<p>For these tasks, <code>sed</code> will become handy for us again!</p>



<p>To remove all the lines <em>before</em> a given line in a file, you&#8217;ll have to use this code:</p>



<p><code>sed -n '/[the pattern itself]/,$p'</code></p>



<p>And to remove all the lines <em>after</em> a given line in a file, this is the code:</p>



<p><code>sed -n '/[the pattern itself]/q;p'</code></p>



<p><em>Note: this one will remove the line with the pattern, too!</em></p>

<p>Let&#8217;s replace the <code>[the pattern itself]</code> parts in your <code>sed</code> commands with the patterns we&#8217;ve found above &#8212; and then add them to your command using pipes.</p>



<p>Something like this:</p>



<pre class="wp-block-preformatted">curl https://www.ted.com/talks/sir_ken_robinson_do_schools_kill_creativity/transcript |

<p><em>Note 1: I used line breaks in my command&#8230; but only to make my code nicer. Using line breaks is optional in this case.</em></p>

If you want to add them, you can. But you&#8217;ll have to know that </em><code><em>*</em></code><em> is a special character in </em><code><em>sed</em></code><em>, so to refer to it as a character in your text, you&#8217;ll have to &#8220;escape&#8221; it with a backslash first. The code would look like this: </em><code><em>sed -n '/\*\*\*\* Programs &amp;amp. initiatives \*\*\*\*/q;p'</em></code><br><em>Again, this won&#8217;t be needed anyway.</em></p>

<h2><strong>Removing first and last lines with the head and tail commands</strong></h2>



<p>And as a final step, remove the first (<code>Details About the talk</code>) and the last (<code>TED</code>) lines of the content you currently see in Terminal! These two lines are not part of the original talk&#8230; and you don&#8217;t want them in your analyses.</p>



<p>For this little modification, let&#8217;s use the <code>head</code> and <code>tail</code> commands (I wrote about them <a href="https://data36.com/data-coding-101-introduction-bash/" data-wpel-link="internal">here</a>).</p>



<p>To remove the last line, add this code: <code>head -n-1</code></p>



<p>And to remove the first line, add this code: <code>tail -n+2</code></p>



<p>And with that, this will be your final code:&nbsp;</p>



<pre class="wp-block-preformatted">curl https://www.ted.com/talks/sir_ken_robinson_do_schools_kill_creativity/transcript |
html2text |
sed -n '/Details About the talk/,$p' |
sed -n '/Programs &amp;amp. initiatives/q;p' <strong>|
head -n-1 |
tail -n+2</strong></pre>



<p>You can try it out&#8230;</p>



<p>But I recommend to save this into a file first, so you will be able to reuse the data you got in the future.</p>



<pre class="wp-block-preformatted">curl https://www.ted.com/talks/sir_ken_robinson_do_schools_kill_creativity/transcript |
html2text |
sed -n '/Details About the talk/,$p' |
sed -n '/Programs &amp;amp. initiatives/q;p' |
head -n-1 |
tail -n+2 <strong> &gt;&nbsp;proto_text.csv</strong></pre>
